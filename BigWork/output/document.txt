Code generation models can translate natural language to source code. These models are often based on large language models like GPT. Evaluating the robustness of code generation is important for reliability. Robustness refers to the ability to handle small changes in input. Perturbations can be applied to the input prompts to test robustness. Common perturbations include typos, paraphrasing, and changes in formatting. The pass@k metric measures the success rate of generated code. Another metric is the robustness pass@k which considers worst-case performance. Fine-tuning with augmented data can improve model robustness. Future work should focus on better evaluation benchmarks.
